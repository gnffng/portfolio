# 프로젝트 요약
생산의 품질과 관련된 데이터를 프로세스에 따라 연쇄적으로 처리해주는 ScQMS 시스템을 개발하고
TTF(한국산업기술시험원)의 Good Software 품질 인증을 받는 프로젝트
# 프로젝트 진행과정
- API : Spring Boot
- UI : React
- OAuth : Spring Oauth2 -> JWT
- DB : MSSQL / QueryDSL
- CI/CD : Docker / Jenkins
- DataGrid : DevExtreme
- PM 대리 직무를 수행하며 약 300여 건의 Pull Request merge.
# 기술 스택
spring / react / mssql / oauth2 / queryDsl / devExtreme / docker / jenkins

# 후기
적은 연차로 PM 직무를 수행하여서 부담이 되기도 하였지만 좋은 기회라고 생각하고 열심히 임했습니다.
부족했던 Git 테크닉을 많이 기를 수 있었고(roleback, rebase 등) Jenkins와 Docker를 다뤄보며 CI/CD를 익혀볼 수 있는 좋은 기회였습니다.
기존에도 동료들의 코드를 리뷰해주고는 했지만, 작정하고 Pull Request를 전담하여 Review하게 되니 동료들과 끊임없이 소통해야 했습니다.
회의에 참석하던 입장에서 회의를 잡는 입장이 되니 한 번의 회의를 잡더라도 최대한 개발자들의 개발 시간을 덜 뺐을 수 있도록 노력했습니다.
이 프로젝트를 통해 Hard Skill도 Soft Skill도 많이 성장할 수 있었던 것 같습니다.
