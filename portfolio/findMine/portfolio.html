# 프로젝트 요약
'React' 프레임워크를 공부하며 토이프로젝트로 지뢰찾기 카피 사이트를 만들었습니다.
(원본사이트 : https://www.google.com/search?q=%EC%A7%80%EB%A2%B0%EC%B0%BE%EA%B8%B0)

# 프로젝트 진행과정
- 기존 사이트에서 리소스를 수집한다.
- 데이터 흐름도를 작성하고 객체의 계층설계를 한다.
- 반응형이라는 패러다임에 어긋나지 않도록 설계한다.
- 'React' 프레임워크를 이용해 웹사이트를 제작한다.

# 기술 스택
jsx / react / css

# 팀원
*주한봉(개인프로젝트)
-> DB설계 / 로직설계 / 코드작성 / 디버깅

# 후기
구조지향, 객체지향에 익숙해져 있던 저에게 반응형이라는 패러다임은 어색하게 느껴졌습니다.
코드를 작성하면서도 몇번이고 '이게 과연 Reactive 한 걸까?' 하는 의문과 함께 멍하니 고민했던 것 같습니다.
중간에 서너차례 설계를 갈아엎기도 했습니다.
새로운 패러다임을 배우는 것은 언제나 쉽지 않은 것 같습니다.
더 다양한 패러다임에 익숙해질 수 있도록 앞으로 더 노력해야겠습니다.
