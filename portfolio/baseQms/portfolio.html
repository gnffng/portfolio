# 프로젝트 요약
생산의 품질과 관련된 기능을 데이터의 흐름에 따라 연쇄적으로 처리해주는 ScQMS 시스템을 웹페이지로 제작함

# 프로젝트 진행과정
- API는 Spring Boot 기반으로 프로젝트를 진행한다.
- UI는 React 기반으로 프로젝트를 진행한다.
- MSSQL을 DB로 사용한다.
- Business Losic에 맞게 DB를 설계한다(범용성 있게).
- 공통코드, 메뉴정보, 메뉴권한 등을 설계하고 Data를 Insert한다.
- QueryDsl을 사용하여 동적 쿼리를 사용한다.
- DataGrid는 DevExtreme을 사용하여 그린다.
- OAuth 서버를 사용하여 token을 발행한다.
- Eureka / Zuul / Hystrix 등의 MSA 설계는 option으로 제공한다.
- token은 JWT로 발행하여 프론트엔드에서 관리한다.

# 기술 스택
spring / react / mssql / oauth2 / queryDsl / devExtreme

# 후기
기존에 있던 QMS 솔루션을 분석하여 GS인증을 받을 수 있는 프로젝트로 리펙토링을 하는 차세대 프로젝트입니다.
UI를 ThymeLeaf에서 React로 변경했기도 하고 GS인증의 기준에 따라 새로 개발할 부분들도 많았기 때문에 리펙토링 보다는 참고자료가 있는 완전 새로운 프로젝트를 개발하는 기분이었습니다.
이 프로젝트를 하며 팀에서 처음으로 PullRequest 문화를 도입하기도 했고 제대로 된 협업을 처음으로 느껴본 프로젝트였기 때문에 많은 경험을 쌓을 수 있었습니다.
역할 분담을 하여 각자 맡은 화면을 구현하는 와중에 공용으로 사용될법한 공용기능들을 범용성있게 개발하여 팀원들에게 공유하는 등 서로의 협업이 잘 이루어졌던 것 같습니다.
저도 Custom IdGenerator나 Email 모듈 등 공용기능을 개발하여 세미나에서 공유를 했던 기억이 납니다.
또한 MSA의 기반 설계를 경험해보며 대용량 트래픽에 대한 고민을 많이 해보는 시간이었던 것 같습니다.
이 프로젝트를 통해 저에게 개발자로서 자신감이 많이 생기게 되었다고 생각합니다.
