# 프로젝트 요약
회사 선배들이 개발하여 주성에 서비스중인 QMS 프로젝트의 유지보수.

# 프로젝트 진행과정
- 매달 들어오는 유지보수 List를 받아서 버그수정, 고도화 등 유지보수를 수행함
# 기술 스택
spring boot / mssql / thymeleaf / oauth2 / queryDsl / devExtreme

# 팀원
*주한봉
-> 쿼리 튜닝 / 버그 Fix / 성능개선 / 테스트
*그외
-> 그외 유지보수 다수

# 후기
회사에 처음 들어온 신입들이 스쳐가는 유지보수 프로젝트입니다.
이 프로젝트에 관련해서는 작은 자랑거리가 하나 있습니다.
처음 입사 1주차에 사수가 이 프로젝트를 분석하며 감을 익히라고 프로젝트를 공유해주셨습니다.
그리고 페이지 설명을 해주시며 상당히 자주 사용하는 페이지의 조회 속도가 1분 이상 걸리는걸 보여주시며 해당 프로젝트의 고질병이라고 소개하셨습니다.
저는 처음 받은 프로젝트를 정말 열심히 해보고 싶었고, 이 프로젝트의 고질병에 포커스를 맞춰서 프로젝트를 분석하였습니다.
그리고 2주차가 되고 저는 쿼리튜닝, NO-OFFSET 기반의 조회로 API를 수정하여 해당 페이지의 조회속도를 7초 내외로 단축시킬 수 있었습니다.
회사의 첫 프로젝트에서 자신감을 얻을 수 있어서 좋았던 기억으로 남는 프로젝트입니다.
