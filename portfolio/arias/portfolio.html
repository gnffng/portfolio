삼성 설계 점검 시스템 SM 개발

Description

  • 주한봉 외 1명

  • 2024.01~2024.05


# 프로젝트 요약
삼성전자의 설비 점검 시스템 SM 운영 개발 프로젝트.

# 프로젝트 진행과정
- API : Spring Boot
- UI : Vue.js
- DB : Oracle
- 협업툴 : Jira
- Tomcat Thread Dump를 Tracing 하여 DB 데드록 문제 해결
- Tomcat Heap Dump를 Tracing 하여 메모리 누수 문제 해결
- 기존에는 동일 사업부서 내에서만 호환되던 데이터를 타 사업 부서에서도 호환되도록 확장 개발
- 유지보수 개발 건 약 25건 처리(쿼리 작성, Data 전처리 등)

# 기술 스택
Spring Boot / Vue.js / Oracle / Jira / Jmap / Jhat

# 후기
기존에 개발이 된 시스템을 SM 운영 개발 하는 것이기 때문에 개발보다는 분석에 훨씬 많은 시간을 들여야 했습니다.
기존 설계 의도에 어긋나게 개발된 부분들이나, 애초부터 설계가 잘못된 부분들을 마주할 때면 항상 '최선의 구조'가 무엇인지 고민했습니다.
당장 리팩토링을 할 여유가 되지 않더라도 개선방향에 대해서는 짬짬히 고민했던 것 같습니다.
잘못된 구조를 모두 바로잡지는 못했지만, 그래도 Scheduler, Tomcat OOME Dump 생성 등 핵심 기능 한 둘 정도는 구조 개선을 할 수 있었습니다.
